Why are the JSONL logs wrong?

Claude Code writes JSONL entries during streaming, before the API response completes. Input tokens get a placeholder value of 0 or 1. When the real number arrives, nobody goes back to update the log. Output tokens exclude thinking/reasoning tokens, which are 60-70% of Opus output.

This was independently verified by Magnus Gille, who found input tokens undercounted by 100-174x across multiple days of heavy use.
